# 모아북 위젯 기획안

> 작성일: 2026-04-14
> 기준 버전: 1.0.18

---

## 컨셉: "모아북 서재"

내 서재에서 매일 한 권을 골라 **책 표지 + 저장 이유(reason)**를 홈화면에 보여주는 위젯.
"왜 이 책을 담았는지" 매일 리마인드해주는 구조.

- **목표**: 저장 동기 강화
- **설명**: 읽고 싶은 책이나 저장 이유를 폰 홈화면에 상시 노출. 앱을 열지 않아도 매일 보이는 구조. 예쁜 위젯은 인스타 공유로 자연스러운 바이럴 효과도 기대할 수 있음.

---

## 위젯 타입 (3종)

### 1. Small (2x2) — 미니 카드

```
┌──────────────┐
│  ┌────┐      │
│  │표지│ 제목  │
│  │    │ ...  │
│  └────┘      │
│ "저장 이유"   │
└──────────────┘
```

- 책 표지 썸네일 + 제목 1줄 + 저장 이유 1줄
- 탭하면 앱의 해당 책 상세(BookInfo)로 이동

### 2. Medium (4x2) — 명언 카드 스타일

```
┌────────────────────────────┐
│                            │
│  ┌────┐  "이 책을 담은 이유"│
│  │표지│                    │
│  │    │  저장 이유 전체 텍스트│
│  └────┘                    │
│         ── 책 제목 / 저자   │
│              모아북 서재     │
└────────────────────────────┘
```

- 인스타 공유에 최적화된 비율
- DungGeunMo 폰트 + 앱 테마 컬러 적용
- **공유 버튼** → 위젯 캡처 이미지를 인스타 스토리/피드로 공유

### 3. Large (4x4) — 내 서재 미니뷰

```
┌────────────────────────────┐
│       모아북 서재            │
│  ┌────┐ ┌────┐ ┌────┐     │
│  │표지│ │표지│ │표지│     │
│  │ 1  │ │ 2  │ │ 3  │     │
│  └────┘ └────┘ └────┘     │
│  ┌────┐                    │
│  │표지│     12권 저장 중    │
│  │ 4  │                    │
│  └────┘                    │
└────────────────────────────┘
```

- 최근 저장한 책 3~4권의 표지를 격자로 표시
- 하단에 "N권 저장 중" 카운트
- 각 표지 탭 → 해당 책 상세 이동
- 서재 전체를 한눈에 보여주는 컬렉션 감성

## 핵심 동작

| 항목 | 설명 |
|------|------|
| 데이터 소스 | `MyBookRepository.getStoreBooks()` API 호출 → 로컬 캐시 |
| 갱신 주기 | 매일 자정 + 앱 실행 시 (WorkManager periodic) |
| 책 선택 로직 | 랜덤 but 최근 7일 내 보여준 책 제외 (SharedPreferences) |
| 미로그인 시 | "모아북에서 첫 책을 담아보세요" 안내 + 앱 열기 유도 |
| 책 0권일 때 | "읽고 싶은 책을 담아보세요" + 검색 화면으로 이동 |

---

## 차별화 포인트

| 요소 | 설명 |
|------|------|
| reason 노출 | 다른 독서앱 위젯에 없는 기능. "왜 담았는지"를 매일 상기시켜 독서 동기 부여 |
| 레트로 디자인 | DungGeunMo 폰트 + 픽셀 보더의 앱 테마를 위젯에도 적용. 인스타 공유 시 브랜드 인지도 |
| 공유 = 바이럴 | Medium 위젯의 공유 버튼 → 인스타 스토리 최적화 사이즈로 캡처. "이 책 담은 이유" 자체가 콘텐츠 |
| 매일 다른 책 | 자정마다 랜덤 교체 (최근 7일 내 중복 방지). 위젯을 볼 때마다 새로운 느낌 |

## 기술 스택

- **Jetpack Glance** (Compose 기반 위젯 API, minSdk 26 OK)
- **WorkManager** — 주기적 데이터 갱신
- **Coil** — 위젯 내 표지 이미지 로딩
- **Hilt** — WorkManager DI 연동

```
moabook/
├── widget/                          ← 신규 모듈

│   │   ├── WidgetContent.kt         # Small/Medium/Large UI 정의
│   │   ├── WidgetTheme.kt           # 위젯용 컬러/폰트
│   │   ├── WidgetDataWorker.kt      # WorkManager로 데이터 갱신
│   │   └── WidgetShareUtil.kt       # 인스타 공유 헬퍼
│   └── src/main/res/
│       └── xml/widget_info.xml      # 위젯 메타데이터
├── domain/                          ← 기존 (변경 없음)
├── data/                            ← 위젯 캐시용 SharedPreferences 추가
└── app/                             ← widget 모듈 의존성 추가
```

## 구현 단계

| Phase | 작업 | 예상 |
|-------|------|------|
| **1** | 모듈 생성 + Glance 기본 Small 위젯 (정적 데이터) | 1일 |
| **2** | WorkManager 연동 + API 데이터 갱신 + 캐시 | 1일 |
| **3** | Medium 위젯 + 테마 스타일링 (DungGeunMo 폰트 등) | 1일 |
| **4** | 딥링크 연동 (위젯 탭 → 책 상세) | 0.5일 |
| **5** | 인스타 공유 기능 (위젯 캡처 → Share Intent) | 1일 |
| **6** | Large 위젯 (서재 미니뷰) | 1일 |
| **7** | 엣지 케이스 처리 (미로그인, 0권, 오프라인) | 0.5일 |

## 인스타 공유 플로우

1. Medium 위젯의 공유 버튼 탭
2. 앱 Activity 실행 → 위젯과 동일한 디자인의 Compose 화면 렌더링
3. `Canvas` → `Bitmap` 캡처
4. `Intent.ACTION_SEND` + `"com.instagram.android"` 패키지로 공유
5. 인스타 미설치 시 일반 공유 시트 fallback
