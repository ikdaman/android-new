# 모아북 디자인 리뷰 계획서

> 작성일: 2026-04-20
> 기준 버전: 1.0.17
> 목적: 위젯 확장 작업 전 전체 UX/UI 건강도 점검
> 산출물: `docs/DESIGN_REVIEW_REPORT.md` (기획자 + 개발자 열람용)

---

## 1. 리뷰 목표

- 모아북의 **레트로 픽셀 브랜드 정체성**이 전 화면에서 일관되게 구현되고 있는지 점검
- **정보 구조·시각 위계·타이포그래피·레이아웃·컬러**의 영향도 높은 개선점을 도출
- 기획자와 개발자가 **즉시 실행 가능한 액션 아이템** 형태로 정리

---

## 2. 리뷰 컨텍스트 (인터뷰 결과)

### 2.1 제품 정체성

| 항목 | 내용 |
|---|---|
| 타겟 사용자 | 전 연령대 (모든 독서 애호가) |
| 핵심 가치 | "내가 읽은 책을 레트로 감성으로 기록하고 공유할 만한 느낌으로" |
| 톤앤매너 | 80~90년대 컴퓨터 / 픽셀·도트 감성 |
| 브랜드 폰트 | DungGeunMo (픽셀 폰트, 컨셉과 정합) |

### 2.2 변경 가능성 매트릭스

| 요소 | 고정 여부 | 리뷰 대상 |
|---|---|---|
| 앱 로고/아이콘 | **고정** | ❌ 리뷰 제외 |
| DungGeunMo 폰트 | **고정** (교체 불가) | ⚠️ 활용도·위계만 리뷰 |
| 화면 구조 / IA | **고정** (탭 구성·내비게이션 유지) | ⚠️ 네이밍·라벨링은 별도 축 |
| 컬러 팔레트 | 변경 가능 | ✅ |
| 간격·레이아웃 세부 | 변경 가능 | ✅ |
| 미세 인터랙션·애니메이션 | 변경 가능 | ✅ |
| 일러스트·아이콘 그래픽 | 변경 가능 | ✅ |

### 2.3 기술 제약

- **최소 지원 API**: Android 10 (API 29)
- **target SDK**: 36 / **compile SDK**: 36
- **다크모드**: 미지원 (리뷰 범위 제외)
- **태블릿/폴더블**: 대응 대상 아님 (폰 전용으로 리뷰)

### 2.4 데이터 상태

| 상태 | 리뷰 계정 |
|---|---|
| 내 서점 (저장 책) | 5권 |
| 독서 히스토리 | 4권 |
| 빈 상태 (empty state) | **스킵** — 별도 캡처하지 않음 |

### 2.5 참고 자료

- Figma 원본: **없음** (실제 캡처 화면만으로 진행)
- 벤치마크 앱: **없음** (절대 기준으로 평가)

### 2.6 사용자가 이미 인지한 문제

> **홈 → "독서 시작" → 히스토리 이동 플로우**
> - 책이 홈 목록에서 사라진 것처럼 느껴짐
> - "히스토리"라는 네이밍 때문에 "읽는 중"인데도 "다 읽은 책"으로 오인됨

→ `/critique` 단계에서 **정보 구조 신호·네이밍·시각적 상태 표시** 관점으로 깊게 분석 (IA는 유지하되 네이밍·메타포 개선안 제안)

## 3. 리뷰 대상 화면

### 3.1 포함 (13개)

| 플로우 | 화면 |
|---|---|
| 진입 | SplashScreen, LoginScreen, SignupScreen |
| 메인 | MainScreen (탭 컨테이너), HomeScreen, HistoryScreen, SettingScreen |
| 책 탐색 | SearchBookScreen, BookInfoScreen, MyBookSearchScreen |
| 책 추가 | AddBookScreen, BarcodeScreen, ManualBookInputScreen |

### 3.2 제외

- TestScreen (개발용)
- 빈 상태 (empty state)
- 위젯 관련 (별도 프로젝트로 분리)
- 친구 관련 기능 (구현되지 않음)

### 3.3 상태 변이 (선별적 캡처)

리뷰 품질을 위해 아래 상태는 필요 시 추가 캡처:

- 로딩 중 상태 (스켈레톤/스피너)
- 에러 상태 (네트워크 실패 등)
- 데이터 있음 vs 적음 (히스토리 4권 상태의 시각적 밀도)

---

## 4. 캡처 전략

### 4.1 도구

- **adb** (디바이스: `R5CRB2A38HM` 연결 확인됨)
- `adb exec-out screencap -p` → PNG 저장
- `adb shell uiautomator dump` → 요소 좌표 추출
- `adb shell input tap/swipe` → 자동 내비게이션

### 4.2 캡처 위치

```
images/review/
├── 01_splash.png
├── 02_login.png
├── 03_home.png
├── ...
└── variants/
    ├── history_loading.png
    └── search_error.png
```

### 4.3 자동화 범위

- **자동**: 탭 이동, 스크롤, 상세 화면 진입
- **수동 개입 필요**: 계정 로그아웃/재로그인, 책 추가 등 상태 변경 액션
- **전제 조건**: 사용자가 **홈 화면**에 앱을 두고 시작 (로그인 완료 상태)

```

[평가 단계 — impeccable 스킬]
 ③  /critique   — UX·정보구조·시각 위계·인지 부하
                 ★ "홈 → 히스토리" 네이밍 이슈 집중 분석
 ④  /typeset    — DungGeunMo 활용도·위계·가독성
 ⑤  /layout     — 간격·정렬·리듬·그리드 일관성
 ⑥  /colorize   — 팔레트·대비·감정 톤 (변경 가능 영역)

[마감 검증]
 ⑦  /polish     — 정렬·일관성 마감 디테일
 ⑧  /audit      — 접근성·반응형 기술 점검

[산출 단계]
 ⑨  통합 리포트 작성
```

### 5.1 단계별 산출물

| 단계 | 산출물 |
|---|---|
| ① 크롤링 | `images/review/*.png` |

| ③~⑥ 평가 | 화면별 개선 카드 (Before 이미지 + 문제 + 근거 + 개선안) |

| ⑧ audit | 접근성·반응형 스코어 리포트 |
| ⑨ 통합 | `docs/DESIGN_REVIEW_REPORT.md` |

## 6. 우선순위 기준

**영향도 우선** 축으로 정렬:

1. **영향도 (High/Med/Low)** — 사용자가 자주 보는 화면 + 핵심 플로우
2. **컨셉 정합성 (High/Med/Low)** — 레트로 픽셀 브랜드와의 부합 정도
3. **공수 (Quick Win / Medium / Deep Work)** — 참고용, 정렬에는 반영 안 됨

### 표시 예시

| 이슈 | 화면 | 영향도 | 컨셉 | 공수 |
|---|---|---|---|---|
| 히스토리 네이밍 오인 | Home, History | 🔴 High | 🟡 Med | Quick Win |

## 7. 최종 리포트 구성 (`DESIGN_REVIEW_REPORT.md`)

**기획자와 개발자 모두가 읽기 쉽도록** 아래 구조로 작성:

1. **Executive Summary** — 3~5개 핵심 발견, 영향도 순
2. **스코어카드** — 각 축(critique/typeset/layout/colorize/audit)별 점수와 요약
3. **화면별 상세** — 각 화면마다:
   - Before 스크린샷
   - 문제 (What) + 근거 (Why) + 개선안 (How)
   - 영향도·컨셉·공수 태그
4. **크로스컷팅 이슈** — 여러 화면에 반복되는 패턴 문제
5. **우선순위 액션 리스트** — 영향도순 정렬, 기획자·개발자 담당 구분
6. **부록** — 인터뷰 원본, 캡처 메타데이터

## 10. 예상 결과물 예시

### 10.1 개선 카드 형식 (샘플)

```markdown
### 🔴 High | History 네이밍 오인 유도

**화면**: HomeScreen → HistoryScreen 전환 시
**문제**: "독서 시작" 이후 책이 홈에서 사라지고 "히스토리"로 이동 →
사용자가 "다 읽은 책" 영역으로 인식해 "내가 지운 건가?" 혼란
**근거**: 사용자 직접 보고 ("책이 사라진 것만 같음")
**개선안**:
  1. 네이밍 변경: "히스토리" → "읽는 중" 또는 "독서 중"
  2. 탭 아이콘에 진행 중 상태 시각 신호 추가 (북마크·체크 아이콘)
  3. 홈에서 "독서 시작" 시 **전환 애니메이션**으로 이동 감각 제공
**공수**: Quick Win (네이밍만 바꿔도 체감 큼)
```
