# 작업목록

1. 내 서점(HomeScreen)에서 체크박스를 클릭하면, 하단에서 책 추가할때와 유사한 Bottom Dialog를 띄워줘.
   - 타이틀은 "이 책을 시작할께요."
   - 그 밑에는 독서 시작, 독서 종료 날짜를 입력하는 UI가 있으면 된다. -> 책 추가화면의 히스토리 UI를 참고해.
     - 근데 독서 시작이랑 독서 종료 날짜를 수정할 수는 없고, 독서 시작 날짜에 오늘 날짜가 들어가고, 독서 종료 날짜에는 "읽는 중"이 들어가면 된다.
   - 그리고 나서 하단에 있는 저장 버튼을 누르면, PATCH /mybooks/{mybookId}/reading-status API를 수행해서 readingStatus를 "READING"으로 넣는다.

2. 책 추가하기 화면에서 뒤로가기를 빠르게 두번 누르지 않도록 막아줘. 이 버튼 외에도 뒤로가기 버튼들에는 다 적용해줘. Modifier.oneClick() 에 딜레이 주는 방식으로 해줘.
3. 설정 화면은 (닉네임)님, 안녕하세요 :) 문구가 상단에 있고, 그 하단에 내 정보 관리를 제목으로 그 밑에 닉네임, 그리고 그에 닉네임을 수정하는 UI를 만들고, inputField 우측에 pencil 모양을 클릭하면 수정 API를 호출하도록 해. 여기서도 중복호출 똑같이 체크 (회원가입 화면처럼)
    - 그리고 그 밑에 회색 선좀 넣어주고, 공지사항, 서비스 이용 약관, 개인정보 처리방침 내용을 넣어줘. 공지사항은 아직 보류야. ~/dev/Android 프로젝트에 서비스 이용 약관이랑 개인정보 처리방침 기능 확인해서 여기도 동일하게 구현해줘. 아마 URL 브라우저로 여는 동작일꺼야.
4. 내 서점 화면에 책 목록이 없을 때 읽고 싶은 책을 추가해보라는 UI를 가운데 넣어줘. 그걸 누르면 책 추가로 이동되게끔 해주고.
5. 책 정보 내용으로 들어가서 수정 버튼을 눌렀을 때 동작도 구현해줘.
   - 책 등록 Bottom Dialog랑 거의 비슷하게 구현하면 되는데 타이틀은 "책 정보 수정"이야.
   - 아직 읽지 않아서 "내 서점"인 상태면 "책 등록"일 때랑 동일하게 UI를 그려
   - "히스토리" 탭이면 독서 시작, 독서 종료 UI 밑에 읽고 싶은 이유도 작성할 수 있도록 추가해줘.
   - 그리고 이 책 정보가 직접 입력한 책이라면 bookInfo.source 값에 "CUSTOM"인데, 이 때는 "책 정보 수정" 화면이 좀 더 입력을 많이 해야 해.
     - 직접 입력해서 책을 추가했던 데이터들을 확인해서 이 "책 정보 수정" Dialog를 완성해줘.
   - 책 정보 수정 API를 수정해줘. 예시 값은 아래와 같아. 수정한 뒤에 책 정보 수정 Dialog에서 저장 버튼을 눌렀을 때 API 호출이 되고 완료되면 내 서점 화면으로 이전 화면으로 되돌아가게 해줘.
     ```json
{
"status": "HISTORY", // or "STORE"
"reason": "읽고싶은 이유",
"historyInfo": {
"startedDate": "2025-03-01T00:50:16Z",
"endedDate": "2025-04-01T00:50:16Z" // 읽는 중으로 변경하는 경우, "endedDate": null
},
"bookInfo": {
"title": "책제목",
"author": "작가",
"publisher": "출판사",
"publishDate": "yyyy-MM-dd", //출간일
"ISBN": "9123456783111", //(ISBN은 13자리 숫자)
"totalPage": 500 // 전체 페이지 수
}
}
```
